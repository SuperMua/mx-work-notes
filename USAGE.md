# MX工作便签使用说明

本文档面向实际使用人员、测试人员和交付验收人员，说明前台和后台的主要使用方式、典型路径与常见问题。

## 1. 登录与注册

### 1.1 前台入口

前台地址通常为：

```text
http://127.0.0.1:5173
```

如果是交付预览模式，则通常为：

```text
http://127.0.0.1:4173
```

未登录访问业务页时，系统会自动跳转到 `/login`。

### 1.2 前台登录

前台登录页支持同页切换“登录”和“注册”两种模式。

登录时需要填写：

- 邮箱
- 密码

登录成功后：

- 如果你原本访问的是某个业务页，会自动返回原目标页
- 如果没有指定目标页，默认进入工作台

### 1.3 前台注册

注册时需要填写：

- 姓名
- 邮箱
- 密码

注册成功后会自动登录并进入系统。

### 1.4 后台登录

后台地址通常为：

```text
http://127.0.0.1:5174
```

如果是交付预览模式，则通常为：

```text
http://127.0.0.1:4174
```

后台只支持登录，不提供注册入口。

后台只有以下角色可以进入：

- `owner`
- `admin`

以下角色会被拒绝访问：

- `editor`
- `viewer`

## 2. 用户端使用说明

## 2.1 工作台

登录后默认进入工作台。

工作台主要用于：

- 查看当前工作概览
- 快速进入便签页
- 查看重点事项
- 进入看板、日历和设置页面

首页的目标是“工作面”，因此操作重点都集中在当前任务和后续动作上。

## 2.2 便签页

便签页是前台最核心的工作区域，路径通常为：

```text
/notes
```

页面采用固定的主从结构：

- 左侧是便签列表
- 右侧是详情预览区

### 左侧列表

左侧列表用于快速扫描和切换记录，支持：

- 搜索
- 按状态筛选
- 按优先级筛选
- 按标签筛选
- 排序

点击任意一条便签后，右侧会立即显示对应详情。

### 右侧详情区

右侧详情区默认是预览状态，不是直接编辑状态。

详情区通常包含以下信息：

- 标题
- 状态
- 优先级
- 截止时间
- 正文内容或清单内容
- 标签
- 创建时间与更新时间

如果需要修改内容，请点击“编辑”按钮打开编辑层。

### 选中逻辑

- 首次进入页面时，系统通常会自动选中当前可见列表中的第一条记录
- 如果当前选中项被删除、归档或被筛选条件排除，系统会自动切换到下一条可用记录
- 如果列表为空，右侧会显示空状态

## 2.3 新建和编辑便签

便签编辑通过弹层完成，不会直接在详情区内联编辑。

你可以创建：

- 普通便签
- 清单便签

编辑时可设置的内容通常包括：

- 标题
- 正文
- 清单项
- 状态
- 优先级
- 截止时间
- 提醒时间
- 标签
- 颜色
- 是否置顶

### 清单便签

如果当前便签是清单类型，你可以勾选子项完成情况。

系统会根据清单完成情况自动推导父项完成状态。

## 2.4 看板

路径通常为：

```text
/board
```

看板用于按状态组织便签，适合拖拽调整任务进度。

主要使用方式：

- 查看不同状态列中的便签
- 拖拽便签到其他状态列
- 结合筛选器快速定位任务

## 2.5 日历

路径通常为：

```text
/calendar
```

日历页面用于查看带有截止时间的便签。

可用于：

- 查看某一天安排了哪些便签
- 按日期扫描未来任务
- 调整便签日期安排

只有设置了截止时间的便签会出现在日历里。

## 2.6 设置

路径通常为：

```text
/settings
```

设置页是用户偏好中心，通常包含以下部分：

- 工作流默认项
- 主题预览
- 通知状态
- 标签风格库

### 主题切换

系统默认提供两套完整主题：

- 浅色工作台
- 深色工作台

主题切换不是只改按钮颜色，而是会联动：

- 页面背景
- 列表
- 详情区
- 表单
- 标签
- 弹层

## 2.7 导入与导出

用户端支持 JSON 导入与导出。

### 导入

导入流程通常为：

1. 打开导入弹层
2. 粘贴或加载 JSON 数据
3. 查看右侧预览摘要
4. 查看冲突提示和导入策略
5. 确认导入

### 导出

导出用于：

- 备份当前数据
- 在不同环境之间迁移数据

## 2.8 归档、回收站与删除

便签支持完整的状态流转：

- 活跃
- 已归档
- 回收站

常见处理方式：

- 不再活跃但要保留：归档
- 临时移除：放入回收站
- 确认不再需要：永久删除

## 2.9 移动端使用节奏

移动端不会强行压缩桌面双栏布局，而是采用更适合手机的节奏：

- 先看列表
- 再进入整屏详情
- 需要修改时再进入整屏编辑层

返回时不会丢失原先的筛选条件和列表状态。

## 3. 后台使用说明

后台是完整的管理端，不是只读演示页。

常见导航包括：

- 数据看板
- 账号管理
- 工作区管理
- 成员管理
- 便签管理
- 模板管理
- 标签管理
- 同步监控

## 3.1 数据看板

数据看板用于快速掌握全局运营情况。

你可以在这里查看：

- 账号总数
- 工作区总数
- 便签总数
- 模板与标签概览
- 已完成与逾期情况
- 同步冲突情况
- 工作区风险排行
- 高频标签
- 最近活动

看板上的模块通常都支持点击直达对应管理页，并自动带上筛选条件。

## 3.2 账号管理

账号管理页用于管理系统用户。

常见操作包括：

- 查看账号列表
- 搜索账号
- 新增账号
- 编辑姓名和邮箱
- 重置密码
- 删除账号

删除限制：

- 如果账号仍然拥有某个工作区的 `owner` 身份，则不能直接删除
- 必须先转移工作区归属或删除其拥有的工作区

## 3.3 工作区管理

工作区管理页用于管理全局工作区。

常见操作包括：

- 查看工作区列表
- 新增工作区
- 编辑工作区信息
- 删除工作区
- 查看工作区拥有者、成员数、便签数、模板数、标签数

从工作区详情通常可以一跳进入：

- 成员管理
- 便签管理
- 模板管理
- 标签管理
- 同步监控

## 3.4 成员管理

成员管理页以工作区为上下文。

你可以：

- 查看某个工作区下的成员列表
- 邀请成员
- 修改成员角色
- 移除成员

角色仍使用四档体系：

- `owner`
- `admin`
- `editor`
- `viewer`

## 3.5 便签管理

便签管理页用于从后台全局管理业务便签。

支持的典型操作：

- 按工作区筛选
- 按标签筛选
- 按关键字搜索
- 按状态筛选
- 按优先级筛选
- 查看详情
- 编辑便签
- 归档
- 放入回收站
- 恢复
- 永久删除

从工作区、标签和看板风险块进入时，系统会自动带上预过滤条件。

## 3.6 模板管理

模板管理页用于统一管理工作区模板。

典型操作包括：

- 新建模板
- 编辑模板
- 删除模板
- 按工作区查看模板

## 3.7 标签管理

标签管理页用于管理工作区标签。

常见能力：

- 查看标签列表
- 修改标签名称
- 修改标签颜色
- 删除标签
- 查看标签使用次数
- 查看标签所属工作区

标签联动规则：

- 标签改名后，会同步更新相关便签的标签数组
- 标签删除后，会从相关便签中移除

## 3.8 同步监控

同步监控页用于查看同步异常和冲突情况。

通常可查看：

- 冲突记录
- 风险工作区
- 与冲突相关的便签或工作区

## 4. 典型使用路径

## 4.1 普通用户日常使用

1. 打开前台登录页。
2. 使用注册或登录进入系统。
3. 在工作台查看今日重点。
4. 进入便签页创建新任务。
5. 点击左侧列表项查看右侧详情。
6. 点击编辑按钮修改标题、清单、状态或截止时间。
7. 在看板或日历中继续管理任务。
8. 在设置页切换主题或调整偏好。

## 4.2 管理员查看全局状态

1. 打开后台登录页。
2. 使用 `owner` 或 `admin` 账号登录。
3. 在数据看板查看总体指标、风险工作区和活动流。
4. 点击某个风险工作区进入便签管理页。
5. 结合工作区、标签和状态筛选定位问题记录。
6. 进入成员或工作区页调整权限与结构。

## 4.3 验收人员快速检查

1. 使用默认账号登录前台。
2. 新建一条便签。
3. 编辑便签内容并设置标签和截止时间。
4. 归档该便签，再放入回收站并恢复。
5. 登录后台。
6. 打开数据看板、账号管理、工作区管理和便签管理页面。

## 5. 常见问题

### 5.1 前台打不开业务页

可能原因：

- 尚未登录
- API 未启动
- 前台环境变量配置错误

### 5.2 登录后马上又被踢回登录页

可能原因：

- API 无法访问
- JWT 配置异常
- 后端服务未正常启动

### 5.3 后台登录成功但无权限进入

可能原因：

- 当前账号角色不是 `owner` 或 `admin`
- 数据库中该账号未分配后台可用角色

### 5.4 默认账号无法使用

请确认是否执行过：

```bash
corepack pnpm db:seed
```

### 5.5 主题切换后没有变化

请确认：

- 当前页面已完成加载
- 本地缓存没有异常
- 页面不是旧构建产物

## 6. 默认账号

默认验收账号如下：

```text
owner@example.com
password123
```

该账号可用于：

- 前台登录
- 后台登录
- 验收基础功能

## 7. 建议的验收顺序

建议交付验收时按以下顺序操作：

1. 先登录前台，确认工作台和便签页可进入。
2. 新建、编辑、归档、删除一条便签。
3. 查看看板、日历、设置页是否正常。
4. 再登录后台。
5. 打开数据看板、账号管理、工作区管理、便签管理。
6. 检查角色边界、数据联动和基础 CRUD 是否正常。
