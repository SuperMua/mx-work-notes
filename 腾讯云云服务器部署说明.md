# MX工作便签腾讯云云服务器部署说明

本文档适用于把 MX工作便签部署到腾讯云环境，采用“CVM + 腾讯云数据库 PostgreSQL + Nginx + HTTPS”的正式交付方案。

## 1. 推荐架构

推荐结构如下：

- 前台：部署在 CVM 上的 Nginx 静态目录，域名例如 `app.example.com`
- 后台：部署在 CVM 上的 Nginx 静态目录，域名例如 `admin.example.com`
- API：部署在 CVM 上的 Node 服务，域名例如 `api.example.com`
- 数据库：腾讯云数据库 PostgreSQL

如果你后续希望前台和后台拆到 COS 或其他静态托管平台，可以在 API 保持不变的前提下调整。

## 2. 腾讯云资源建议

建议至少准备：

- 1 台 CVM，建议 `2 核 4G` 或更高
- 1 个腾讯云数据库 PostgreSQL 实例
- 1 套域名
- 1 套 SSL 证书

建议原则：

- CVM 与数据库位于同一地域
- 数据库优先使用私有网络连接
- 对外只暴露 80/443，API 的 3000 不直接暴露公网

## 3. 对应产品建议

- 服务器：CVM
- 数据库：腾讯云数据库 PostgreSQL
- 证书：腾讯云 SSL 证书
- 反向代理与静态站点：Nginx

如果后续需要前后端分发加速，可以再叠加 CDN。

## 4. 购买与网络规划建议

### 4.1 CVM 建议

推荐：

- 系统：Ubuntu 22.04 LTS 或 24.04 LTS
- 放通端口：`22`、`80`、`443`
- 不对外开放 `3000`

### 4.2 数据库建议

推荐：

- 选择与 CVM 同地域
- 放在同一 VPC
- 启用自动备份
- 使用独立数据库账号

### 4.3 私网访问原则

建议让 API 通过私网访问腾讯云数据库 PostgreSQL：

- 更安全
- 延迟更低
- 不需要暴露数据库公网入口

## 5. 服务器初始化

登录 CVM 后，先准备基础环境：

```bash
sudo apt update
sudo apt install -y git nginx
```

然后安装 Node.js 20 运行环境，最终确保：

```bash
node -v
corepack --version
```

可正常使用即可。

## 6. 拉取项目

建议目录：

```text
/srv/mx-work-notes
```

执行：

```bash
git clone git@github.com:SuperMua/mx-work-notes.git /srv/mx-work-notes
cd /srv/mx-work-notes
```

## 7. 环境变量配置

复制模板：

```bash
cp apps/api/.env.example apps/api/.env
cp apps/web/.env.example apps/web/.env
cp apps/admin/.env.example apps/admin/.env
```

### 7.1 API 环境变量

示例：

```bash
DATABASE_URL=postgresql://dbuser:dbpassword@postgres-private-host:5432/smart_notes?schema=public
JWT_SECRET=请替换为强随机密钥
PORT=3000
NODE_ENV=production
```

### 7.2 前后台环境变量

前台：

```bash
VITE_API_BASE_URL=https://api.example.com
```

后台：

```bash
VITE_API_BASE_URL=https://api.example.com
```

## 8. 初始化数据库

执行：

```bash
corepack pnpm install
corepack pnpm db:generate
corepack pnpm db:migrate
```

如果你在验收环境需要默认账号，可额外执行：

```bash
SEED_OWNER_EMAIL=admin@example.com SEED_OWNER_PASSWORD=强密码 corepack pnpm db:seed
```

注意：

- `NODE_ENV=production` 生效后，seed 会跳过
- 正式环境不建议保留默认密码

## 9. 构建与发布

构建：

```bash
corepack pnpm build
```

复制前后台静态产物：

```bash
sudo mkdir -p /var/www/mx-web
sudo mkdir -p /var/www/mx-admin
sudo cp -r apps/web/dist/* /var/www/mx-web/
sudo cp -r apps/admin/dist/* /var/www/mx-admin/
```

## 10. API 启动建议

推荐使用 `systemd` 守护 API。

文件：

```text
/etc/systemd/system/mx-api.service
```

示例：

```ini
[Unit]
Description=MX Work Notes API
After=network.target

[Service]
Type=simple
WorkingDirectory=/srv/mx-work-notes
Environment=NODE_ENV=production
ExecStart=/usr/bin/node /srv/mx-work-notes/apps/api/dist/apps/api/src/main.js
Restart=always
RestartSec=5
User=www-data
Group=www-data

[Install]
WantedBy=multi-user.target
```

启用：

```bash
sudo systemctl daemon-reload
sudo systemctl enable mx-api
sudo systemctl start mx-api
sudo systemctl status mx-api
```

## 11. Nginx 配置示例

```nginx
server {
    listen 80;
    server_name app.example.com;
    root /var/www/mx-web;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}

server {
    listen 80;
    server_name admin.example.com;
    root /var/www/mx-admin;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}

server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 12. HTTPS 与证书建议

推荐使用腾讯云 SSL 证书服务完成证书签发，然后加载到：

- Nginx
- 或腾讯云 CLB / 网关

正式上线前必须确保：

- 前台 `https`
- 后台 `https`
- API `https`
- 前端构建变量已写成 `https://api.example.com`

## 13. 腾讯云数据库 PostgreSQL 注意事项

建议：

- 与 CVM 保持同地域、同私网
- 启用备份
- 使用数据库白名单或网络控制
- 如需公网连接，再评估 SSL 加密

如果你打算开启数据库 SSL，可参考腾讯云数据库 PostgreSQL 的 SSL 文档进行配置。

## 14. 上线检查

至少检查：

- `https://api.example.com/health` 返回正常
- 前台和后台登录页可访问
- 正常账号可登录
- 可以创建、编辑、归档、删除便签
- 后台数据看板可打开
- 如果执行过 seed，默认密码已改
