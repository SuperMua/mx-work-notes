# MX工作便签阿里云云服务器部署说明

本文档适用于把 MX工作便签部署到阿里云环境，采用“阿里云云服务器 ECS + ApsaraDB RDS for PostgreSQL + Nginx + HTTPS”的正式交付方案。

## 1. 适用方案

推荐架构如下：

- 前台：部署到 ECS 上的 Nginx 静态目录，域名例如 `app.example.com`
- 后台：部署到 ECS 上的 Nginx 静态目录，域名例如 `admin.example.com`
- API：部署到 ECS 上的 Node 服务，域名例如 `api.example.com`
- 数据库：ApsaraDB RDS for PostgreSQL

如果你后续还想把前台和后台拆到 OSS 或 CDN，可以在 API 保持不变的前提下再调整。

## 2. 阿里云资源建议

建议至少准备：

- 1 台 ECS 云服务器，建议 `2 核 4G` 或更高
- 1 个 ApsaraDB RDS for PostgreSQL 实例
- 1 个或多个域名
- 1 套 SSL 证书

建议资源保持：

- ECS 和 RDS 位于同一地域
- ECS 和 RDS 尽量放在同一 VPC
- 对外只开放 Nginx 的 80/443，API 的 3000 仅本机访问

## 3. 推荐的阿里云产品对应关系

- 云服务器：ECS
- 数据库：ApsaraDB RDS for PostgreSQL
- 证书：SSL 证书管理服务
- 负载与转发：Nginx 或阿里云负载均衡

如果是单机交付，Nginx 已经足够；如果后续要做多机扩展，再考虑负载均衡。

## 4. 购买与网络规划建议

### 4.1 ECS 选择建议

建议：

- 系统：Ubuntu 22.04 LTS 或 24.04 LTS
- 磁盘：系统盘足够即可，静态资源占用不大
- 安全组：先开放 `22`、`80`、`443`

### 4.2 RDS 选择建议

建议：

- PostgreSQL 版本选择当前稳定版本
- 与 ECS 同地域
- 使用独立账号和独立数据库
- 打开自动备份

### 4.3 内网互通原则

推荐让 ECS 和 RDS 通过内网通信，这样更稳妥：

- 延迟更低
- 不暴露数据库公网入口
- 更利于安全组和白名单控制

## 5. 服务器初始化

登录 ECS 后，建议先准备：

```bash
sudo apt update
sudo apt install -y git nginx
```

然后安装 Node.js 20 运行环境。你可以使用自己熟悉的方式安装，只要最终满足：

```bash
node -v
corepack --version
```

都能正常使用即可。

## 6. 拉取项目与目录建议

建议将项目放到：

```text
/srv/mx-work-notes
```

执行：

```bash
git clone git@github.com:SuperMua/mx-work-notes.git /srv/mx-work-notes
cd /srv/mx-work-notes
```

## 7. 环境变量配置

复制环境变量模板：

```bash
cp apps/api/.env.example apps/api/.env
cp apps/web/.env.example apps/web/.env
cp apps/admin/.env.example apps/admin/.env
```

### 7.1 API 环境变量建议

示例：

```bash
DATABASE_URL=postgresql://dbuser:dbpassword@rds-private-endpoint:5432/smart_notes?schema=public
JWT_SECRET=请替换为强随机密钥
PORT=3000
NODE_ENV=production
```

### 7.2 前后台环境变量建议

前台：

```bash
VITE_API_BASE_URL=https://api.example.com
```

后台：

```bash
VITE_API_BASE_URL=https://api.example.com
```

## 8. 数据库初始化

在 ECS 上执行：

```bash
corepack pnpm install
corepack pnpm db:generate
corepack pnpm db:migrate
```

如果你在验收环境确实需要默认账号，可临时执行：

```bash
SEED_OWNER_EMAIL=admin@example.com SEED_OWNER_PASSWORD=强密码 corepack pnpm db:seed
```

注意：

- 如果 `NODE_ENV=production` 已经生效，`db:seed` 会跳过
- 正式环境不要长期使用默认密码

## 9. 构建与发布

执行：

```bash
corepack pnpm build
```

构建产物：

- 前台：`apps/web/dist`
- 后台：`apps/admin/dist`
- API：`apps/api/dist/apps/api/src/main.js`

将前后台发布到静态目录：

```bash
sudo mkdir -p /var/www/mx-web
sudo mkdir -p /var/www/mx-admin
sudo cp -r apps/web/dist/* /var/www/mx-web/
sudo cp -r apps/admin/dist/* /var/www/mx-admin/
```

## 10. 启动 API

建议使用 `systemd` 托管 API。

创建：

```text
/etc/systemd/system/mx-api.service
```

示例：

```ini
[Unit]
Description=MX Work Notes API
After=network.target

[Service]
Type=simple
WorkingDirectory=/srv/mx-work-notes
Environment=NODE_ENV=production
ExecStart=/usr/bin/node /srv/mx-work-notes/apps/api/dist/apps/api/src/main.js
Restart=always
RestartSec=5
User=www-data
Group=www-data

[Install]
WantedBy=multi-user.target
```

启用：

```bash
sudo systemctl daemon-reload
sudo systemctl enable mx-api
sudo systemctl start mx-api
sudo systemctl status mx-api
```

## 11. Nginx 配置建议

推荐 3 个站点：

- `app.example.com`
- `admin.example.com`
- `api.example.com`

示例：

```nginx
server {
    listen 80;
    server_name app.example.com;
    root /var/www/mx-web;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}

server {
    listen 80;
    server_name admin.example.com;
    root /var/www/mx-admin;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}

server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 12. HTTPS 与证书建议

在阿里云环境中，推荐：

- 证书统一托管到阿里云 SSL 证书管理服务
- 再部署到 ECS 或由 Nginx 手动加载

正式上线前，确保：

- 前台和后台走 `https`
- API 也走 `https`
- 前台构建时写入的 `VITE_API_BASE_URL` 是 `https://api.example.com`

## 13. RDS 安全建议

建议启用：

- 数据库备份
- 内网访问优先
- IP 白名单控制
- 如有公网访问需求，再评估 SSL 加密

如果你的 RDS 需要启用 SSL 连接，请以阿里云官方文档为准配置 `DATABASE_URL` 和客户端证书。

## 14. 上线检查

上线前请至少检查：

- ECS 能访问 RDS
- `https://api.example.com/health` 返回正常
- 前台和后台登录页可打开
- 使用正式账号能登录
- 能创建、编辑、归档、删除便签
- 如果执行过 seed，默认密码已修改

## 15. 推荐的更新流程

```bash
cd /srv/mx-work-notes
git pull
corepack pnpm install
corepack pnpm db:generate
corepack pnpm db:migrate
corepack pnpm build

```
